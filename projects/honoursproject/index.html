Honours Project

I undertook my honours during my fourth year of university. In my research project, I wanted to experiment with different types of dataset collection and preprocessing methodologies and how they affected the performance of Topic models (a type of AI model).

  • The goal of a topic model is to summarise a long piece of text/s by examining the co-occurrence patterns of words and organising the words it sees into a ’topic'.
  • For example, if a topic model received news articles as an input, it would probably group words like “climate”, “temperature”, and “renewable” together and words / phrases like “economy”, “interest rates”, and “housing prices” together. This is because these words frequently co-occur together. Topic models have shown utility when applied to texts to auto ’tag’ them or to organise texts.

There are emerging ideas about using topic models on social media posts from customers / target markets to extract common topics that are interesting to them. Although, topic models have performance issues when applied to social media data. Because the length of a social media post is so short, when a topic model tries to obtain co-occurrence patterns between words, it generally gets bad quality outcomes because of the lack of data.

Therefore, my research project’s aim was to improve the quality of the topics output by a topic model when applied to social media-based data. To do this, I would need to add more information or context to a given social media post. So, when scraping social media posts from the internet, I didn’t just scrape the content of the post, but I also checked if the user had shared any websites and then visited that site, scraped the data of the site and concatenated it to the social media post. I then put this data into a topic mode. I compared the results to a topic model that only took the social media posts themselves as input - i.e. did not receive the extra contextual information from the shared websites.

The result was a resounding success! The model with the extra information from the hyperlinked websites produced higher quality topics than the models without. Below I outlined a broad summary of the code written for this project and the results.

Scraping and Processing the Social Media Posts with Python

To gather data for my project, I built a web scraper in Python that took tweets from X/Twitter and deposited them into an excel file.

As inputs, the program takes in the number of tweets wanted, and a hashtag to get tweets from a broad topic.

The program accesses the stream of tweets being posted to X/Twitter live in a particular hashtag and stops once it has accessed the number defined by the user.

import snscrape.modules.twitter as sntwitter
import pandas as pd
import requests
import re


class TwitterScraper:
    query = "placeholder"
    tweet_num = 1


    def scrape(self, query, tweet_num):
        self.query = query
        self.tweet_num = int(tweet_num) - 1
        tweets = []
   
        for i, tweet in enumerate(sntwitter.TwitterHashtagScraper(query).get_items()):
            if i>int(tweet_num):
                break
            tweets.append([tweet.date, tweet.id, tweet.content, tweet.user.username, tweet.retweetCount,
            tweet.replyCount, tweet.likeCount, tweet.quoteCount, tweet.lang, tweet.media, tweet.url])
            if tweet.media != None:
                self.download_image(tweet.media, i)
   
        tweets_df = pd.DataFrame(tweets, columns=['Datetime', 'Tweet_Id', 'Text', 'Username',
        'Retweet_Count', 'Reply_Count', 'Like_Count', 'Quote_Count', 'Language', 'media', 'URL'])
        tweets_json = tweets_df.to_json()


        with open(r'C:\Users\FILEPATH...', 'w', encoding="utf-8") as f:        
            tweets_df.to_csv(f, header=['Datetime', 'Tweet_Id', 'Text', 'Username',
            'Retweet_Count', 'Reply_Count', 'Like_Count', 'Quote_Count', 'Language', 'media', 'URL'])

Social media posts are multimedia. To only download the text component of a post would remove important context to the post. Unfortunately, the Twitter API does not separate the png file name for us, so this has to be done manually with regex in the ’extract_file_name’ method. Once the media url is obtained, it can be downloaded. When giving the file a name when downloading it, it is important to write code as seen in the ‘download image’ method to give each image a unique name that also associates the image with its corresponding tweet. This is so that after, the text and images can be combined in a singular dataframe easily later on.

    def download_image(self, tweet_media_list, index):
        images_in_tweet = 0
        for i in tweet_media_list:
            if self.is_photo(str(i)):
                media_url = self.extract_file_name(str(i))
                file_name = r'C:\Users\FILEPATH...' + str(index) + '_' + str(images_in_tweet) + '.png'
           
                img_data = requests.get(media_url).content
                with open(file_name, 'wb') as handler:
                    handler.write(img_data)
                images_in_tweet = images_in_tweet + 1
   
    def extract_file_name(self, media_string):
        media_url = media_string
        media_url = re.search('fullUrl=\'.+\'', media_url).group()
        media_url = media_url.replace('fullUrl=\'', '')
        media_url = media_url.replace('\'', '')
        return media_url


    def is_photo(self, media_string):
        return media_string[0] == 'P'

To scrape the data from the hyperlinked websites shared by users in the tweets, a similar program was written. Except, as the websites shared by users were all different, I could not access an API like before to easily access all the information. Thus, I built a html parser and then scraped the body text once parsed. The code to do this resembled the following:

Once the dataset was downloaded, the text of the tweets were cleaned by removing special characters such as newlines, return characters, slashes, etc. Other characters can also be removed such as @ signs and # signs.

def clean_extracted_media(s): # where s is a string of scraped html
    if s is not None:
        s = re.sub(r'\s+', ' ', s)
        s = re.sub('\r+', '', s)
        s = re.sub('\r', '', s)
        s = re.sub('\n+', '', s)
        s = re.sub('\n', '', s)
        s = re.sub("\'", '', s)
        s = re.sub('#', '', s)
        return s

Building the models

Building the models - the corpus ‘without_hyperlink’ does not contain the data scraped from the shared websites and the corpus ‘with_hyperlink’ does have the data scraped from the shared websites.

The model / algorithm used for this project is the ‘Latent Dirichlet Allocation’ model or LDA.

Results

The following table shows a sample of the topics (a topic consists of a group of related words) output by both models. The topics from the model without the hyperlinked documents were susceptible noise (random words, misspellings and single letters). Models generally become susceptible to noise when there is too little data or too much meaningless data. In this case, the model only using data from the tweets has too little data to work from so it essentially ‘sees’ patterns that aren’t there. On the other hand, the model with the data from the shared websites created higher quality topics with greater coherence.

Because there is so little text to train on with the model that is only trained on tweet text, the topics output by this model classified strings such as twitter handles (eg. ‘wallacemomjaime’) as important. With the other model that is trained on both the tweet text and the shared website text, it had enough data to identify trends. For example, this model identified topics related to luxury / designer fashion, anime / cosplay / halloween (the data was scraped in the later half of October so lots of people were tweeting about halloween). The topics from the other model do not identify any sort of trends.
